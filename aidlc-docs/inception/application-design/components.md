# コンポーネント定義（Components）

## 1. CsvGenerator（CSV疑似データ生成）
**責務**: `要件定義.md` / `requirements.md` FR-1に基づき、A→B→C→D→Eの生産フローに沿った疑似データCSVを生成する。
- 稼働日カレンダー（日曜休/月曜9:00開始/休憩時間帯）を考慮した加工時間の生成
- ボトルネック工程（加工機C）基準の1日理論生産数を計算し、その範囲内でランダムに完成品数・号機割当を決定
- 同一製品のA〜E工程間で時系列の整合性（後工程は前工程完了後に開始）を保証
- 低確率のNG判定・設備停止相当（2〜3倍時間）データの注入
- **インターフェース**: 対象日を受け取り、`データ転送前/A1_ProcessingMachine_Utilization_Rate.csv` を出力する

## 2. DbLoader（CSV → SSMS 取り込み）
**責務**: `データ転送前/` のCSVを読み込み、型変換のうえSSMSへMERGE取り込みする（FR-2）。
- `Product_Number` + `Machine_Number` による重複防止MERGE
- 参照負荷軽減のためのインデックス設計・活用
- 取り込み結果に応じたファイル移動（成功: `データ転送後/`、失敗: `エラー項目/`）
- **インターフェース**: CSVファイルパスを受け取り、取り込み結果（成功/失敗、件数）を返す

## 3. UtilizationCalculator（稼働率算出）
**責務**: SSMSのデータを参照し、日別/週別/週間想定/月別の稼働率を算出する（FR-3）。
- 三交代制・日曜休・月曜補正を考慮した計画時間の算出
- 号機ごとの実加工時間集計、標準加工時間を用いた想定稼働率算出
- **インターフェース**: 対象期間（日/週/月）を受け取り、号機別の稼働率データ（表形式）を返す

## 4. DashboardApp（Streamlit WEB表示）
**責務**: `UtilizationCalculator` の結果をBIツール風ダッシュボードとして表示する（FR-4）。
- 日別（折れ線・号機別色分け）/週別(実績+想定・破線)/月別（表）の3ビュー
- 号機ごとの表示/非表示トグルボタン
- 折れ線グラフのツールチップ（ホバーで値表示）
- **インターフェース**: ブラウザからアクセスされるWebアプリケーション（`streamlit run`）

## 5. SchedulingOrchestration（定期実行オーケストレーション）
**責務**: GitHub Actions（Self-hosted Runner）から12時間毎に `CsvGenerator` → `DbLoader` → （必要に応じて）`UtilizationCalculator` の更新処理を順番に実行する（FR-5）。
- ワークフロー定義（`.github/workflows/*.yml`）の管理
- 実行順序: ①CSV生成・DB取込 → ②稼働率算出更新
- **インターフェース**: GitHub Actionsワークフローとしてスケジュール起動（cron）
