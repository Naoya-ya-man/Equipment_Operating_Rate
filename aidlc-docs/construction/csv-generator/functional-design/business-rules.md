# ビジネスルール（UOW-1: CsvGenerator）

## BR-1: 実行窓（ExecutionWindow）の決定
- GitHub Actionsの実行時刻を基準に、12時間の実行窓（例: 00:00〜12:00, 12:00〜24:00）を決定する
- 実行窓が属する日が**日曜日**の場合、この窓では一切のデータを生成しない（`available_seconds = 0`）
- 実行窓が**月曜日**を含む場合、0:00〜9:00の時間帯を非稼働として除外する

## BR-2: 稼働可能秒数（available_seconds）の算出
- `available_seconds = 窓の長さ(秒) - 窓内に含まれる休憩時間(秒) - 窓内に含まれる非稼働時間(秒、月曜0-9時等)`
- 休憩時間帯: 10:00-10:20, 12:15-13:15, 15:00-15:15, 18:00-18:45, 21:00-21:15, 2:00-2:45, 5:00-5:15

## BR-3: 理論生産数上限（この窓あたり）の算出
- 加工機タイプごとに `1号機あたりの理論処理可能数 = available_seconds ÷ 標準加工時間(秒)` を算出し、号機数(5)を掛ける
- ボトルネック工程（基準値では加工機C）の値を、この窓における完成品数の理論上限とする
- **この値はあくまで目安（上限）であり、実際の生成数はこれを超えない範囲でランダムに決定する**

## BR-4: 実際の完成品数の決定（故障・不良を考慮）
- 実際に生成する完成品チェーン数は、理論上限に対して低確率の設備停止（BR-7）・不良判定（BR-8）による実効生産数の低下を織り込み、理論上限よりやや少なめの範囲でランダムに決定する
- 曜日・時間帯による増減は許容する（固定値を使わない）

## BR-5: 製品チェーンの時系列整合性
- 同一 `ProductChain` 内で、後工程の開始時刻は前工程の終了時刻以降とする（A完了→B開始→…→E完了）
- 各工程の開始時刻は、休憩時間帯と重ならないように調整する（休憩時間帯に加工が開始・またがることはない）

## BR-6: 実行窓をまたがない制約（重要）
- 1つの `ProductChain` は、開始から完了まで**同一の実行窓内で完結**させる。実行窓をまたぐ製品チェーンは生成しない
- 新規チェーンを開始する際、標準加工時間の合計（A+B+C+D+E、休憩時間帯回避や設備停止の可能性を見込んだ安全マージンを含む）が `window_end` までに収まる場合のみ開始する。収まらない場合はその窓でのチェーン生成を打ち切る
- **【2026-07-09修正】各加工機タイプの5号機は並行して稼働する**。新規チェーンの開始可否は「いずれかのA号機が空く時刻」を基準に判定し、1製品分（約130分）の処理が完了するまで次の製品の生成を待つ、といった直列（1本のラインのような）実装にはしない。この点は当初の実装で誤っており（BR-3の理論生産数が5号機の並行稼働を前提に算出されているにもかかわらず、生成ロジックが直列だったため、実際の生成数が理論値を大幅に下回っていた）、`UnitAssignmentTracker`が号機ごとに「次に利用可能になる時刻」を個別に管理する方式に修正した

## BR-7: 設備停止相当（低確率のブレークダウン）
- 各工程（ProcessingRecord）につき、約**1%の確率**で標準加工時間の2〜3倍（ランダム）の時間を要する「設備停止相当」のデータとする
- 発生した場合でもBR-6の制約（実行窓内で完結）を優先する。ブレークダウンにより窓を超えそうな場合は、その工程の時間を窓の残り時間に収まる範囲にクリップする

## BR-8: 不良判定（NG判定）
- 各 `ProcessingRecord` につき、約**3%の確率**で `pass_judgment = "NG"`、それ以外は `"OK"` とする
- NG判定は各工程で独立に判定する（前工程がNGでも後工程は生成する。歩留まり分析用のデータであり、生産フロー自体は継続する前提）

## BR-9: 号機（1〜5）の均等配分
- 同一 `MachineType` 内での号機割当は、`assignment_count` が均等になるようラウンドロビン的な要素を加味したランダム化を行う
- 特定の号機に偏った割当を避ける
- 割当回数が同数の号機が複数ある場合は、最も早く空く（利用可能になる）号機を優先する（BR-6の並行稼働モデルに対応）

## BR-10: 品番(Product_Number)の採番
- `base_product_id`（6桁）はチェーンごとに一意な連番または乱数で採番する
- 各工程の `product_number` は `base_product_id + machine_name`（末尾1桁）で構成する
