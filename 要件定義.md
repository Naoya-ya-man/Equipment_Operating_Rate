# 製造業における設備稼働率監視ボードの作成

## 目的・使用言語・ツール

1. 製造業における設備の稼働率をリアルタイムで算出
2. CSVに疑似データを作成→Pythonでデータを成形→SSMSに引き上げ
3. PythonによりSSMSのデータを参照し、そのデータを元にしてWEBアプリに稼働率データを表示
4. リアルタイムに動かしたいため、Pythonの一連の実行はGitHubアクションで行う

## 前提

1. 基本的に私に確認するときは日本語でお願いします。

### SSMS接続情報

- データベース: SQL Server (SSMS)
- サーバー名: .\SQLEXPRESS
- データベース名: Equipment_Utilization_Rate
- テーブル名: [dbo].[A1_ProcessingMachine_Utilization_Rate]
- カラム名・データ型:
  ・Product_Number : varchar(50) (品番7桁(1からスタートする))
  ・Machine_Name : varchar(50) (加工機名称(加工機A, 加工機B, 加工機C,加工機D, 加工機E))
  ・Machine_Number : varchar(50)
  ・Processing_Start_Time : datetime
  ・Processing_Completion_Time : datetime
  ・Sum_DateTime : int
  ・Pass judgment : varchar(50)
- 隠すべき情報があれば事前に教えてください。

### Python の作成

- 目的は、CSVの疑似データをSSMSに引き上げるという目的
- データ転送前 にCSVを作成したい。形式は、上記のDBカラムの順番に合わせて作成してほしい。
- CSV名はA1_ProcessingMachine_Utilization_Rate.CSVとする
- データはランダムに作成してほしい、品番は7桁のうちの最後の一桁に加工機末尾のアルファベットを入れてほしい。加工機名称はそのままで、ランダムに使う
- 加工号機もランダムに、それぞれ5号機まで存在する想定。
- 開始時間もランダムに記入、終了時間もランダムに記入。開始から終了までの時間をSum_DateTimeに記入。INTに変換が必要？
- 時間のランダム基準は、Aは加工に約20分, B=30, C= 40, D = 25, E = 15分で終わる想定だが、＋－10分くらいを視野に入れてランダム投入したい。（アクシデントも想定されるため。）また、ごくまれに二倍~三倍のデータを入れてもいい（設備停止が起こるときもあるから）
- ランダムで少ない確率で不良品判定にNGを入れて不良品扱いにしたい。(歩留まりの参考になるため。)基本はOKと記入
- 日曜日は全休日のためCSVを作る必要がないので作成をスキップ。
- 月曜日は0~9:00までは仕事が始まっていないのでそこを考慮した作り方をする
- 休憩時間は10:00~10:20, 12:15~13:15, 15:00~15:15とし、遅番は18:00~18:45, 21:00~21:15,夜勤は2:00~2:45, 5:00~5:15とするため、基本的にはそこに加工時間は入らない
- CSVに記入されたデータをそれぞれのデータ型に合わせて成形を行い、SSMSのSQL文に沿ったPythonでのSQL文の書き方によって、
  対象テーブルにMERGE文で引き上げてほしい。MERGE条件は、テーブルにすでに同じ品番と加工号機のものがないかを確かめて入れるようにする。
- 見に行くデータ量に負荷をかけたくないので、インデックスを採用し,見に行くデータを削減して実行を軽くしたい。
  あるデータに関してはインサートを行わない仕様としたい。
- データ転送前のCSVをDBに変換した後は、データ転送前フォルダからデータ転送後　にCSVを移動させたい。
  仮に何らかのエラーが出た場合は、エラー項目　に移動させる。
- 生産数は全号機合わせて一日80台想定なのでそれを踏まえてランダム投入にしてほしい。12時間に一回実行であれば40:40が好ましいが、曜日や時間、日付、故障にも左右されると思うので
  それも考慮してランダム投入にする。多少多かったり少なかったりは許容していいこととする。

### Pythonの作成②　→ WEB画面に表示

- 目的は、CSVから上げたSSMSのテーブルのデータを引っ張り、稼働率を算出したい。
- 算出したい情報は、

1. 日別稼働率
2. 週間稼働率・週間想定稼働率
3. 月別稼働率

- 週間稼働率に関しては、動いている号機で集計して想定の稼働率を算出する。想定稼働率の求める時間は、CSV作成で定義した加工剛毅たちの標準時間を基にする。
- 製造業は三交代のため、稼働時間は一日(86400秒)とする。日曜日は全休。月曜日は9:00修行のためそれを考慮して86400秒から差し引く

### 稼働率の定義

1. 各号機ごとに、その日の製品の実際の加工時間(総秒数)÷一日の計画時間(86400)×100
2. 各号機ごとに、日～月曜日の一週間の稼働時間を合計÷一週間の計画時間の合計×100
3. 各号機ごとに、日～月曜日の一週間の加工枚数×標準の加工時間÷一週間の計画時間の合計×100

### WEB画面の構成

- BIツールのモーションボードを意識した構成にしたい。
- 折れ線グラフでそれぞれの加工号機を色分けし、日別の表・週別、週別想定稼働率の2つが入っている表・月ごとの表
  の3つを画面に表示させる。想定稼働率は破線にしてください
- 号機が5つあるため、クリックすると号機ごとに表示非表示切り替えができるわかりやすいボタンを表示。
- カーソルを各号機の折れ線に乗せるといつがどれくらいの稼働率なのか見えるようにする。

### GitHubActionの設定

- 定期実行できるようにしたいので、GitHubActionで使える定期実行用のymlフォーマットをください。
  形式としては、CSV→DBに上げるPythonを先に回し、WEB表示用の稼働率計算&表示のPythonを後に実行させたい。実行頻度は、ランダムデータ投入なので12時間に一回実行としたい。

### StreamlitのURL発行→WEB表示

- URLを発行したいので、それを採用すればすぐ使えるフォーマットをください。
